# 🎙️ Sideline Triage Assistant

**AI voice-triage decision support for untrained grassroots sports coaches.**

Built for the Idea2Impact Online Hackathon 2026 — Theme 3: Crisis Management, HealthTech & Emergency Response.

## The Problem

Youth and amateur sports programs have no sideline access to sports-medicine
expertise. When a player goes down, the coach — who almost never has medical
training — has to make a split-second call about how serious the injury is
and what to do next. There's no time for a detailed report, and a wrong call
can make the injury worse.

## The Solution

The coach taps a button, says a few urgent words about what happened (even
fragmented, e.g. *"Number 7, twisted her ankle bad"*), and Gemini's
multimodal audio understanding:

1. Transcribes whatever was said, however brief
2. Extracts the player and the symptom
3. Assigns a risk level — **Low / Moderate / High**
4. Returns a short, actionable "what to do right now" protocol
5. Flags anything it had to assume, given incomplete information

For High-risk results, the UI surfaces a one-tap emergency-call button. The
tool is explicitly framed as **decision support, not medical treatment** —
it never replaces a doctor, paramedic, or emergency services.

## Tech Stack

| Layer | Tech |
|---|---|
| Backend | Python, FastAPI, `anyio` (async thread offload) |
| AI | Google Gemini API (`gemini-3.5-flash`) via `google-genai` SDK — structured JSON output, minimal-thinking mode for low latency |
| Frontend | Single-page HTML/CSS/JS served directly by FastAPI — no build step, uses the browser's native `MediaRecorder` API |
| Deployment | [Render](https://render.com) (or any platform that runs a Python web service) |

**Why Gemini is central, not decorative:** every triage decision — the risk
level, the extracted player/symptom, and the recommended protocol — is
generated by the model directly from the raw audio in a single call. There is
no separate speech-to-text step and no hardcoded rules engine; the model does
transcription, extraction, and clinical-style risk classification together,
constrained by a JSON schema so the output is reliably structured.

## Setup & Run Locally

1. **Clone the repo**
   ```bash
   git clone https://github.com/nishanth-0427/voice-triage-backend.git
   cd voice-triage-backend
   ```

2. **Create a virtual environment & install dependencies**
   ```bash
   python -m venv venv
   venv\Scripts\activate        # Windows
   # source venv/bin/activate   # macOS/Linux
   pip install -r requirements.txt
   ```

3. **Set your Gemini API key** (get a free one at https://aistudio.google.com/apikey)
   ```bash
   set GEMINI_API_KEY=your_key_here        # Windows cmd
   $env:GEMINI_API_KEY="your_key_here"     # PowerShell
   # export GEMINI_API_KEY=your_key_here   # macOS/Linux
   ```

4. **Run the server**
   ```bash
   uvicorn main:app --reload
   ```

5. Open **http://127.0.0.1:8000**, allow microphone access, and record a report.
